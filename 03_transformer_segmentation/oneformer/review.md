## One-Line Summary

OneFormer achieves universal segmentation with a single jointly trained model by conditioning on task tokens and using query-text contrastive learning to distinguish between semantic, instance, and panoptic tasks.

## Motivation and Problem Statement

Mask2Former demonstrated that one architecture works for all segmentation tasks but still required training three separate models (one per task). This is inefficient in terms of training compute, storage, and deployment. OneFormer addresses this by training a single model once that can perform all three tasks, conditioned on a task input token at inference time. The key challenge is preventing multi-task interference where optimizing for one task degrades another.

## Architecture Overview

OneFormer builds on Mask2Former's architecture with two key additions: (1) a task-conditioned joint training strategy that samples one task per training iteration and conditions the model via a task token; (2) a query-text contrastive loss that aligns object queries with text descriptions of their categories, providing additional semantic supervision. The backbone, pixel decoder, and transformer decoder follow Mask2Former's design.

## Technical Details

### Task Token Conditioning

A task token (one of {semantic, instance, panoptic}) is provided as input. This token is used to initialize the N object queries via a task-conditioned query initialization: the task token is processed through an MLP and added to the learnable query embeddings, biasing them toward the target task's output format.

### Query Initialization

Queries are initialized as: `Q = Q_learnable + MLP(task_token)`. This task-dependent initialization allows the same set of queries to behave differently depending on the task — focusing on semantic regions for semantic segmentation and individual instances for instance segmentation.

### Contrastive Loss

A contrastive loss aligns each object query's representation with the text embedding of its assigned class. Text representations are generated by a frozen CLIP text encoder from templates like "a photo of a {class}". This regularization provides additional semantic grounding and helps queries develop class-aware representations.

### Backbone and Decoder

Same as Mask2Former: Swin-L backbone, multi-scale deformable pixel decoder, 9-layer transformer decoder with masked attention and round-robin multi-scale feature consumption.

### Loss Function

Total loss = mask loss (BCE + Dice) + classification loss (CE) + contrastive loss (InfoNCE). The contrastive loss weight is 0.5. During training, one task is sampled per iteration uniformly at random.

## Experiments and Results

### Datasets

ADE20K, Cityscapes, COCO — evaluated on all three segmentation tasks per dataset.

### Key Results

| Task | Dataset | OneFormer (Swin-L) | Mask2Former (Swin-L) |
|------|---------|-------------------|---------------------|
| Semantic | ADE20K | 58.0 mIoU | 57.8 mIoU |
| Panoptic | ADE20K | 49.8 PQ | 48.1 PQ |
| Semantic | Cityscapes | 84.4 mIoU | 83.3 mIoU |
| Panoptic | COCO | 58.0 PQ | 57.8 PQ |
| Instance | COCO | 49.0 AP | 50.1 AP |

OneFormer matches or exceeds Mask2Former on most benchmarks with a single jointly trained model vs. three separate models. Instance AP on COCO is slightly lower but panoptic and semantic improve.

### Ablation Studies

Task conditioning: +1.2 PQ over no conditioning. Contrastive loss: +0.8 PQ. Joint training with task token outperforms separate training on panoptic (+1.7 PQ) and semantic (+0.2 mIoU) while slightly underperforming on instance (-1.1 AP).

## Strengths

- Single model for all three tasks — 3× reduction in training compute and storage
- Task token is a simple, elegant conditioning mechanism
- Contrastive loss provides useful regularization from language supervision
- Matches or exceeds Mask2Former's per-task specialized training

## Limitations

- Slightly lower instance segmentation performance compared to task-specific Mask2Former
- Requires CLIP text encoder during training (adds dependency)
- Task token must be specified at inference — no automatic task detection
- Joint training is slower per iteration than single-task training

## Connections

Direct evolution of Mask2Former, adding joint training capability. The task token concept relates to prompt conditioning in language models. The contrastive loss bridges vision-language alignment (CLIP) with segmentation. Influenced by multi-task learning literature on avoiding negative transfer.
